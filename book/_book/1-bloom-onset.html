S1.2 Bloom onset

This section details the methodology used to estimate the onset of phytoplankton blooms based on satellite-derived chlorophyll-a time series.

Chlorophyll-a bloom detection

Satellite-derived chlorophyll-a data were processed to identify the onset of seasonal blooms.

  • Load chlorophyll data
  • Calculate sustained bloom start

The chlorophyll-a data is loaded from a CSV file. The data is cleaned by removing missing values (NA), selecting the relevant date and chlorophyll columns, and converting the date column to a proper date format using lubridate::dmy()

CHL_df <- read.csv(here("data", "MODIS.CHL.R2022.csv"), header = TRUE) %>%
  drop_na() %>%
  dplyr::select(date, chlorophyll) %>%
  rename(daily_value = chlorophyll)

CHL_df <- CHL_df %>%
  mutate(date = dmy(date))

Two custom functions were developed to estimate bloom onset dates for different seasonal windows. Both apply the following procedure:

  • Seasonal window: Define a “season year” spanning either December–June or September–June.

  • Baseline: Calculate the seasonal median chlorophyll-a concentration.

  • Thresholds: Set dynamic thresholds at 5%, 15%, 25%, and 30% above the seasonal median.

  • Sustained increase: Identify the first day when two consecutive 7-day rolling medians exceed the threshold, ensuring that the increase is persistent rather than transient.

  • Output: Return the bloom start date, chlorophyll-a concentration on that date, and the corresponding threshold.

  • Winter to Spring window
  • Autumn to Spring Window
  • Summary of bloom start dates

This function defines the season as running from December 21st of the previous year to June 20th of the current year.

calculate_sustained_bloom_start <- function(data_frame, value_column_name = "daily_value", min_duration_days = 7) {

  thresholds <- c(0.05, 0.15,0.25, 0.30)

  # Prepare and annotate data
  data_frame <- data_frame %>%
    mutate(date = as.Date(date)) %>%
    arrange(date) %>%
    mutate(season_year = ifelse(month(date) == 12, year(date) + 1, year(date)))

  # Define seasonal window
  season_ranges <- data_frame %>%
    distinct(season_year) %>%
    mutate(
      season_start_date = ymd(paste0(season_year - 1, "-12-21")),
      season_end_date = ymd(paste0(season_year, "-06-20"))
    )

  # Filter seasonal data
  seasonal_data <- data_frame %>%
    left_join(season_ranges, by = "season_year") %>%
    filter(date >= season_start_date & date <= season_end_date) %>%
    arrange(season_year, date)

  # Compute yearly medians
  medians <- seasonal_data %>%
    group_by(season_year) %>%
    summarise(yearly_median = median(!!sym(value_column_name), na.rm = TRUE), .groups = "drop")

  # Initialize final result with medians
  all_results <- medians
  bloom_date_raws <- list()

  for (thr in thresholds) {
    label <- paste0("thr", thr * 100)

    # Add threshold column
    threshold_tbl <- medians %>%
      mutate(threshold = yearly_median * (1 + thr))

    seasonal_thr_data <- seasonal_data %>%
      left_join(threshold_tbl, by = "season_year")

    # Compute rolling medians
    seasonal_thr_data <- seasonal_thr_data %>%
      group_by(season_year) %>%
      mutate(
        rolling_median_week1 = rollapply(
          !!sym(value_column_name), min_duration_days, median, na.rm = TRUE, fill = NA, align = "left"
        ),
        rolling_median_week2 = rollapply(
          lead(!!sym(value_column_name), n = min_duration_days), min_duration_days, median, na.rm = TRUE, fill = NA, align = "left"
        ),
        cond_i = !is.na(rolling_median_week1) & rolling_median_week1 >= threshold,
        cond_ii = !is.na(rolling_median_week2) & rolling_median_week2 >= threshold
      ) %>%
      filter(cond_i & cond_ii) %>%
      summarise(bloom_date = min(date, na.rm = TRUE), .groups = "drop")

    # Store raw date for difference calculation
    bloom_date_raws[[label]] <- seasonal_thr_data %>%
      rename(!!paste0("bloom_date_raw_", label) := bloom_date)

    # Join back to get bloom values and threshold
    enriched <- seasonal_thr_data %>%
      left_join(seasonal_data, by = "season_year") %>%
      group_by(season_year) %>%
      summarise(
        !!paste0("bloom_start_", label) := format(first(bloom_date), "%d/%m/%Y"),
        !!paste0("value_at_bloom_", label) := seasonal_data[[value_column_name]][match(first(bloom_date), seasonal_data$date)],
        !!paste0("threshold_", label) := medians$yearly_median[medians$season_year == first(season_year)] * (1 + thr),
        .groups = "drop"
      )

    # Join enriched info to results
    all_results <- all_results %>%
      left_join(enriched, by = "season_year")
  }

  # Join all bloom date raws
  bloom_date_df <- reduce(bloom_date_raws, left_join, by = "season_year")

  # Add differences between 5% vs 15% and 30%
  all_results <- all_results %>%
    left_join(bloom_date_df, by = "season_year") %>%
    mutate(
      diff_days_5_vs_15 = as.numeric(bloom_date_raw_thr15 - bloom_date_raw_thr5),
      diff_days_5_vs_30 = as.numeric(bloom_date_raw_thr30 - bloom_date_raw_thr5)
    ) %>%
    select(-starts_with("bloom_date_raw_"))

  return(all_results)
}

S1.3 Visualization of Chlorophyl-a bloom start dates

To examine the distribution and timing of bloom onsets, detection results were transformed and visualized. Density plots compare bloom start dates across thresholds and seasonal windows.

Data processing for visualization

Prior to plotting, results were transformed into long format for analysis with ggplot2. A helper function (process_for_plotting) standardized bloom dates across years and summarized them by detection threshold. Dates from September–December were aligned to 2000 and January–June dates to 2001, creating a continuous seasonal year.

Code 
process_for_plotting <- function(results) {
  results %>%
    select(season_year, starts_with("bloom_start_")) %>%
    pivot_longer(
      cols = -season_year,
      names_to = "threshold_id",
      values_to = "bloom_date_str",
      values_drop_na = TRUE
    ) %>%
    mutate(
      threshold_label = gsub("bloom_start_thr", "", threshold_id),
      threshold = factor(paste0(threshold_label, "%"), levels = c("5%", "15%", "25%", "30%")),
      bloom_date = dmy(bloom_date_str),
      bloom_day_month = if_else(
          month(bloom_date) >= 9,
          update(bloom_date, year = 2000),
          update(bloom_date, year = 2001)
      )
    )
}

Interpretation of Bloom start date figure

This figure shows the frequency distribution of phytoplankton bloom start dates. The left column displays results for a Winter–Spring seasonal window (Dec–Jun), while the right column uses a broader Autumn–Spring window (Sep–Jun). Each row represents a different detection threshold (5%, 15%, 25%, and 30%) relative to the seasonal median chlorophyll-a concentration.

Key patterns include:

Higher Thresholds Delay Bloom Onset: As the detection threshold increases from 5% to 30%, the timing of the bloom onset is consistently pushed later into the year. The earliest blooms, detected with the 5% threshold, peak in December and January. In contrast, the most intense blooms (30% threshold) do not typically start until late January or February. This pattern is expected, as it takes more time for phytoplankton biomass to accumulate to higher concentrations.

Bimodal Onset: The Autumn-Spring analysis reveals a distinct bimodal pattern in bloom timing. An initial wave of blooms occurs in early autumn (September-October), followed by a lull. After this quiet period, the primary, more sustained spring bloom begins in winter, with its subsequent timing and frequency closely mirroring the patterns observed in the Winter-Spring-only analysis. This overall bimodal structure strongly suggests the early autumn blooms are event-driven phenomena, likely triggered by nutrient injection from late-season storms or hurricanes.

Bloom variability: The decreasing sample size (n) for higher thresholds indicates strong year-to-year variability. This suggests that high-magnitude blooms are not an annual occurrence, pointing to the influence of variable environmental conditions.

## Calculate sustained bloom start

Two custom functions were developed to estimate bloom onset dates for different seasonal windows. Both apply the following procedure:

-   Seasonal window: Define a “season year” spanning either December–June or September–June.

-   Baseline: Calculate the seasonal median chlorophyll-a concentration.

-   Thresholds: Set dynamic thresholds at 5%, 15%, 25%, and 30% above the seasonal median.

-   Sustained increase: Identify the first day when two consecutive 7-day rolling medians exceed the threshold, ensuring that the increase is persistent rather than transient.

-   Output: Return the bloom start date, chlorophyll-a concentration on that date, and the corresponding threshold.

:::: panel-tabset
## Winter to Spring window

This function defines the season as running from December 21st of the previous year to June 20th of the current year.

```{r}
#| label: Function to calculate sustained bloom start date

calculate_sustained_bloom_start <- function(data_frame, value_column_name = "daily_value", min_duration_days = 7) {

  thresholds <- c(0.05, 0.15,0.25, 0.30)

  # Prepare and annotate data
  data_frame <- data_frame %>%
    mutate(date = as.Date(date)) %>%
    arrange(date) %>%
    mutate(season_year = ifelse(month(date) == 12, year(date) + 1, year(date)))

  # Define seasonal window
  season_ranges <- data_frame %>%
    distinct(season_year) %>%
    mutate(
      season_start_date = ymd(paste0(season_year - 1, "-12-21")),
      season_end_date = ymd(paste0(season_year, "-06-20"))
    )

  # Filter seasonal data
  seasonal_data <- data_frame %>%
    left_join(season_ranges, by = "season_year") %>%
    filter(date >= season_start_date & date <= season_end_date) %>%
    arrange(season_year, date)

  # Compute yearly medians
  medians <- seasonal_data %>%
    group_by(season_year) %>%
    summarise(yearly_median = median(!!sym(value_column_name), na.rm = TRUE), .groups = "drop")

  # Initialize final result with medians
  all_results <- medians
  bloom_date_raws <- list()

  for (thr in thresholds) {
    label <- paste0("thr", thr * 100)

    # Add threshold column
    threshold_tbl <- medians %>%
      mutate(threshold = yearly_median * (1 + thr))

    seasonal_thr_data <- seasonal_data %>%
      left_join(threshold_tbl, by = "season_year")

    # Compute rolling medians
    seasonal_thr_data <- seasonal_thr_data %>%
      group_by(season_year) %>%
      mutate(
        rolling_median_week1 = rollapply(
          !!sym(value_column_name), min_duration_days, median, na.rm = TRUE, fill = NA, align = "left"
        ),
        rolling_median_week2 = rollapply(
          lead(!!sym(value_column_name), n = min_duration_days), min_duration_days, median, na.rm = TRUE, fill = NA, align = "left"
        ),
        cond_i = !is.na(rolling_median_week1) & rolling_median_week1 >= threshold,
        cond_ii = !is.na(rolling_median_week2) & rolling_median_week2 >= threshold
      ) %>%
      filter(cond_i & cond_ii) %>%
      summarise(bloom_date = min(date, na.rm = TRUE), .groups = "drop")

    # Store raw date for difference calculation
    bloom_date_raws[[label]] <- seasonal_thr_data %>%
      rename(!!paste0("bloom_date_raw_", label) := bloom_date)

    # Join back to get bloom values and threshold
    enriched <- seasonal_thr_data %>%
      left_join(seasonal_data, by = "season_year") %>%
      group_by(season_year) %>%
      summarise(
        !!paste0("bloom_start_", label) := format(first(bloom_date), "%d/%m/%Y"),
        !!paste0("value_at_bloom_", label) := seasonal_data[[value_column_name]][match(first(bloom_date), seasonal_data$date)],
        !!paste0("threshold_", label) := medians$yearly_median[medians$season_year == first(season_year)] * (1 + thr),
        .groups = "drop"
      )

    # Join enriched info to results
    all_results <- all_results %>%
      left_join(enriched, by = "season_year")
  }

  # Join all bloom date raws
  bloom_date_df <- reduce(bloom_date_raws, left_join, by = "season_year")

  # Add differences between 5% vs 15% and 30%
  all_results <- all_results %>%
    left_join(bloom_date_df, by = "season_year") %>%
    mutate(
      diff_days_5_vs_15 = as.numeric(bloom_date_raw_thr15 - bloom_date_raw_thr5),
      diff_days_5_vs_30 = as.numeric(bloom_date_raw_thr30 - bloom_date_raw_thr5)
    ) %>%
    select(-starts_with("bloom_date_raw_"))

  return(all_results)
}
 bloom_startsA <- calculate_sustained_bloom_start(CHL_df)
```

# S1.3 Visualization of Chlorophyl-a bloom start dates

To examine the distribution and timing of bloom onsets, detection results were transformed and visualized. Density plots compare bloom start dates across thresholds and seasonal windows.

### Data processing for visualization

Prior to plotting, results were transformed into long format for analysis with ggplot2. A helper function (process_for_plotting) standardized bloom dates across years and summarized them by detection threshold. Dates from September–December were aligned to 2000 and January–June dates to 2001, creating a continuous seasonal year.

```{r}
#| label: Frequency of bloom start dates
#| code-fold: true

process_for_plotting <- function(results) {
  results %>%
    select(season_year, starts_with("bloom_start_")) %>%
    pivot_longer(
      cols = -season_year,
      names_to = "threshold_id",
      values_to = "bloom_date_str",
      values_drop_na = TRUE
    ) %>%
    mutate(
      threshold_label = gsub("bloom_start_thr", "", threshold_id),
      threshold = factor(paste0(threshold_label, "%"), levels = c("5%", "15%", "25%", "30%")),
      bloom_date = dmy(bloom_date_str),
      bloom_day_month = if_else(
          month(bloom_date) >= 9,
          update(bloom_date, year = 2000),
          update(bloom_date, year = 2001)
      )
    )
}
```

### Interpretation of Bloom start date figure

This figure shows the frequency distribution of phytoplankton bloom start dates. The left column displays results for a Winter–Spring seasonal window (Dec–Jun), while the right column uses a broader Autumn–Spring window (Sep–Jun). Each row represents a different detection threshold (5%, 15%, 25%, and 30%) relative to the seasonal median chlorophyll-a concentration.

Key patterns include:

Higher Thresholds Delay Bloom Onset: As the detection threshold increases from 5% to 30%, the timing of the bloom onset is consistently pushed later into the year. The earliest blooms, detected with the 5% threshold, peak in December and January. In contrast, the most intense blooms (30% threshold) do not typically start until late January or February. This pattern is expected, as it takes more time for phytoplankton biomass to accumulate to higher concentrations.

Bimodal Onset: The Autumn-Spring analysis reveals a distinct bimodal pattern in bloom timing. An initial wave of blooms occurs in early autumn (September-October), followed by a lull. After this quiet period, the primary, more sustained spring bloom begins in winter, with its subsequent timing and frequency closely mirroring the patterns observed in the Winter-Spring-only analysis. This overall bimodal structure strongly suggests the early autumn blooms are event-driven phenomena, likely triggered by nutrient injection from late-season storms or hurricanes.

Bloom variability: The decreasing sample size (n) for higher thresholds indicates strong year-to-year variability. This suggests that high-magnitude blooms are not an annual occurrence, pointing to the influence of variable environmental conditions.
