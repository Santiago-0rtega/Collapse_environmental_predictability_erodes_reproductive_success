Configuring Parallel Processing

To accelerate model fitting, a parallel processing strategy was implemented using the future package. This approach leverages inter-chain parallelization, where each of the model’s independent Markov chains is run simultaneously on a separate CPU core.

The future::plan(multisession, workers=4) command configures four parallel R sessions (workers). When brms is subsequently run with chains = 4, it automatically distributes one chain to each worker. The efficient cmdstanr backend was also selected, substantially reducing overall computation time by executing the chains concurrently rather than sequentially.

options(mc.cores = parallel::detectCores(), brms.backend = "cmdstanr")

Threading can accelerate computation by distributing processes across multiple cores. This requires installing the cmdstanr package and its CmdStan backend.

Model function

A wrapper function was used to fit Gaussian location–scale models for each bloom threshold (5%, 15%, 25%).

run_location_scale_models <- function(response_vars, data) {
  if (!dir.exists(here("Rdata"))) dir.create(here("Rdata"))
  
  purrr::walk(response_vars, function(var) {
    model_formula <- bf(
      as.formula(paste(var, "~ 1 + TIME")),
      sigma ~ 1 + TIME
    )
    
    file_path <- here("Rdata", paste0("model_", var, ".rds"))
    model_family <- gaussian()
    model_priors <- get_prior(model_formula, data = data, family = model_family)
    
    fit <- tryCatch({
      brm(
        formula = model_formula,
        data = data,
        prior = model_priors,
        family = model_family,
        iter = 6000, warmup = 2000, chains = 4, seed = 123, init = 0,
        control = list(adapt_delta = 0.99, max_treedepth = 15),
        file = file_path,
        file_refit = "on_change"
      )
    }, error = function(e) {
      cat("Error fitting model for:", var, "\n")
      cat("Error message:", e$message, "\n")
      return(NULL)
    })
    
    if (!is.null(fit)) cat("Saved model for:", var, "to", file_path, "\n")
  })
}

Hierarchical Bayesian models of reproductive performance

Data preparation

Before fitting the models, the primary dataset was loaded and pre-processed. Separate data frames were created for males and females, with the following key steps:

  • Type conversion: Variables were converted to the appropriate type. Reproductive outcome (FLEDS) was encoded as ordered factors for use in cumulative ordinal models.

  • Centering of continious predictors: TIME, X5MISMATCH,AGE, AFR, LONGEVITY.

  • Identifiers: Year (YEAR) and individual identity (RING) were treated as factors.

  • X5MISMATCH: Phenological mismatch at the 5% bloom threshold (days).

  • FLEDS: Number of fledged chicks (ordinal).

  • ZX5MISMATCH: Centered phenological mismatch (5% threshold).

  • AGE: Age of the individual, centered.

  • AFR: Age at first reproduction, centered.

  • LONGEVITY: Age at last observation (proxy for lifespan), centered.

  • YEAR: Calendar year of observation (factor).

  • RING: Individual identifier (factor).

Models syntax

Two model families were used:

  • Gaussian models for continuous responses (e.g., phenological mismatch).

  • Cumulative ordinal models for ordered categorical responses (e.g., clutch size and fledgling success).

  • Gaussian models (Phenological mismatch)
  • Ordinal models (Number of fledglings)

Automated model fitting workflow

To systematically evaluate candidate models, we developed a custom R function, run_analysis_workflow(). This function automates model fitting and evaluation by:

  • Defining candidate models: Iterating over combinations of fixed and random effects structures.

  • Flexibility: Handling Gaussian and ordinal model families.

  • Modeling auxiliary parameters: Estimating residual standard deviation (sigma) in Gaussian models or the discrimination parameter (disc) in ordinal models.

  • Error handling and efficiency: Skipping models already fitted, saving results automatically, and using tryCatch to prevent failures from halting the workflow.

  • Model evaluation: Saving both the fitted models and their leave-one-out cross-validation (LOO) objects.

  fit_and_save_model <- function(name, mean_random, aux_formula, ...) {
    model_path <- file.path(output_dir, paste0("female_fit_", name, ".rds"))
    loo_path <- file.path(output_dir, paste0("female_loo_", name, ".rds"))

## Configuring Parallel Processing

To accelerate model fitting, a parallel processing strategy was implemented using the `future` package. This approach leverages inter-chain parallelization, where each of the model's independent Markov chains is run simultaneously on a separate CPU core.

The `future::plan(multisession, workers=4)` command configures four parallel R sessions (workers). When `brms` is subsequently run with chains = 4, it automatically distributes one chain to each worker. The efficient cmdstanr backend was also selected, substantially reducing overall computation time by executing the chains concurrently rather than sequentially.


## Model function

A wrapper function was used to fit Gaussian location–scale models for each bloom threshold (5%, 15%, 25%).

```{r}
#| label: bloom-function
#| evalve: false
run_location_scale_models <- function(response_vars, data) {
  if (!dir.exists(here("Rdata"))) dir.create(here("Rdata"))
  
  purrr::walk(response_vars, function(var) {
    model_formula <- bf(
      as.formula(paste(var, "~ 1 + TIME")),
      sigma ~ 1 + TIME
    )
    
    file_path <- here("Rdata", paste0("model_", var, ".rds"))
    model_family <- gaussian()
    model_priors <- get_prior(model_formula, data = data, family = model_family)
    
    fit <- tryCatch({
      brm(
        formula = model_formula,
        data = data,
        prior = model_priors,
        family = model_family,
        iter = 6000, warmup = 2000, chains = 4, seed = 123, init = 0,
        control = list(adapt_delta = 0.99, max_treedepth = 15),
        file = file_path,
        file_refit = "on_change"
      )
    }, error = function(e) {
      cat("Error fitting model for:", var, "\n")
      cat("Error message:", e$message, "\n")
      return(NULL)
    })
    
    if (!is.null(fit)) cat("Saved model for:", var, "to", file_path, "\n")
  })
}
```

# Automated model fitting workflow

To systematically evaluate candidate models, we developed a custom R function, run_analysis_workflow(). This function automates model fitting and evaluation by:

-   Defining candidate models: Iterating over combinations of fixed and random effects structures.

-   Flexibility: Handling Gaussian and ordinal model families.

-   Modeling auxiliary parameters: Estimating residual standard deviation (`sigma`) in Gaussian models or the discrimination parameter (`disc`) in ordinal models.

-   Error handling and efficiency: Skipping models already fitted, saving results automatically, and using `tryCatch` to prevent failures from halting the workflow.

-   Model evaluation: Saving both the fitted models and their leave-one-out cross-validation (`LOO`) objects.

```{r}
    filter(grepl("^ABS", name))
  }
  
  # Safety check
  if (nrow(model_specs) == 0) {
   warning(paste0("No models found for subset key: '", model_subset_key, "'. Check your model names and key."))
   return(invisible(NULL)) # Stop if no models to run
  }
 }
  
  
  # --- Internal function to fit a single model ---
  fit_and_save_model <- function(name, mean_random, aux_formula, ...) {
    model_path <- file.path(output_dir, paste0("female_fit_", name, ".rds"))
    loo_path <- file.path(output_dir, paste0("female_loo_", name, ".rds"))
    
    if (file.exists(model_path)) {
      message(paste0("Skipping '", name, "': Model file already exists."))
      return(invisible(NULL))
    }
    
    message(paste0("\n--- Fitting model: ", name, " ---"))
    
    formula_str <- paste(response_var, "~", fixed_effects, "+", mean_random)
    
    # --- CORRECTED LOGIC ---
    # Use inherits() for a robust check if aux_formula is a real formula or NA.
    brm_formula <- if (!inherits(aux_formula, "formula")) {
      bf(formula_str, nl = FALSE)
    } else if (is_gaussian) {
      bf(formula_str, sigma = aux_formula, nl = FALSE)
    } else {
      bf(formula_str, disc = aux_formula, nl = FALSE)
    }
    
    model_priors <- get_prior(brm_formula, data = data, family = model_family)
    
    fit <- tryCatch({
      brm(
        formula = brm_formula, data = data, prior = model_priors, family = model_family,
        iter = 6000, warmup = 2000, chains = 4, seed = 123, init = 0,backend = "cmdstanr",threads = threading(12),
        control = list(adapt_delta = 0.99, max_treedepth = 15)
      )
    }, error = function(e) { message(paste0("ERROR fitting '", name, "': ", e$message)); NULL })
    
    if (!is.null(fit)) {
      saveRDS(fit, model_path)
      loo_result <- tryCatch(loo(fit), error = function(e) { message(paste0("ERROR with LOO: ", e$message)); NULL })
      if (!is.null(loo_result)) saveRDS(loo_result, loo_path)
    }
  }
  
  # --- Run all models and compare LOO results ---
  pwalk(model_specs, fit_and_save_model)
  

}
```
